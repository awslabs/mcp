# AWS Security Hub MCP Server

import { LinkButton } from '@site/src/components/LinkButton';

Model Context Protocol (MCP) server for AWS Security Hub, enabling security findings analysis, compliance reporting, and security posture management through AI assistants.

## Features

- **Security Findings Analysis**: Retrieve and filter security findings with comprehensive criteria
- **Compliance Reporting**: Get compliance status across enabled security standards
- **Security Insights**: Access both AWS-managed and custom Security Hub insights
- **Finding Statistics**: Analyze security findings with aggregated statistics and trends
- **Workflow Management**: Update finding workflow status and add remediation notes
- **Security Scoring**: Calculate and track overall security posture metrics

### Manual Configuration

```json
{
  "mcpServers": {
    "awslabs.aws-security-hub-mcp-server": {
      "command": "uvx",
      "args": ["awslabs.aws-security-hub-mcp-server@latest"],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR",
        "AWS_PROFILE": "your-aws-profile",
        "AWS_REGION": "us-east-1"
      }
    }
  }
}
```

## Prerequisites

### AWS Requirements

1. **AWS Security Hub enabled** in your AWS account and region
2. **AWS credentials configured** (via AWS CLI, environment variables, or IAM roles)
3. **Required IAM permissions**:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "securityhub:GetFindings",
                "securityhub:GetInsights",
                "securityhub:GetInsightResults",
                "securityhub:BatchUpdateFindings",
                "securityhub:GetComplianceDetails",
                "securityhub:DescribeStandards",
                "securityhub:DescribeStandardsControls",
                "securityhub:GetEnabledStandards"
            ],
            "Resource": "*"
        }
    ]
}
```

## Available Tools

### get_security_findings
Retrieve security findings with comprehensive filtering options including severity levels, workflow status, compliance status, and time ranges.

**Example usage:**
```
Get all critical severity findings from the last 7 days
```

### get_compliance_summary
Get compliance status summary across all enabled security standards and controls.

**Example usage:**
```
Show me the compliance summary for AWS Foundational Security Standard
```

### get_insights
Retrieve Security Hub insights including both AWS-managed and custom insights for trend analysis.

**Example usage:**
```
Show me security insights for identifying patterns in my findings
```

### get_finding_statistics
Get aggregated statistics about security findings with grouping options by severity, product, or resource type.

**Example usage:**
```
Get security finding statistics grouped by severity for the last 30 days
```

### update_finding_workflow
Update the workflow status of security findings and add remediation notes.

**Example usage:**
```
Mark finding with ID "arn:aws:securityhub:us-east-1:123456789012:finding/..." as resolved
```

### get_security_score
Retrieve the overall security score and security posture trends over time.

**Example usage:**
```
What's my current security score and how has it changed?
```

## Configuration

### Environment Variables

- `AWS_PROFILE`: AWS profile to use (optional, defaults to default profile)
- `AWS_REGION`: AWS region to use (optional, defaults to us-east-1)
- `FASTMCP_LOG_LEVEL`: Logging level (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- `SECURITY_HUB_MAX_RESULTS`: Maximum number of results to return per query (default: 100)

## Security Considerations

- All API calls use configured AWS credentials with appropriate IAM permissions
- Findings data may contain sensitive security information - use appropriate access controls
- Consider using AWS IAM roles with temporary credentials in production environments
- The server implements rate limiting and result pagination for AWS API protection

## Troubleshooting

### Common Issues

1. **AccessDenied errors**: Ensure your AWS credentials have the required Security Hub permissions
2. **Security Hub not enabled**: Enable Security Hub in your AWS account and region
3. **No findings returned**: Check if Security Hub has findings in the specified region and time range
4. **Rate limiting**: The server implements appropriate rate limiting for AWS API calls

### Debug Mode

Enable debug logging by setting `FASTMCP_LOG_LEVEL=DEBUG` to see detailed API calls and responses.

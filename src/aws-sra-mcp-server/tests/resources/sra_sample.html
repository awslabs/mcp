AWS Security Reference Architecture (AWS SRA)

The AWS Security Reference Architecture (AWS SRA) is a holistic set of guidelines for deploying the full complement of AWS security services in a multi-account environment. It provides prescriptive guidance on how to architect a security foundation across AWS accounts and AWS Organizations.

Overview

The AWS SRA is designed to help you:

  • Establish a strong security foundation for your AWS environment
  • Implement security services in a consistent and scalable manner
  • Follow AWS security best practices
  • Meet common compliance requirements

Key Components

Security Tooling Account

Centralized account for security services like AWS Security Hub, Amazon GuardDuty, and AWS Config.

Log Archive Account

Dedicated account for storing and analyzing logs from all accounts in the organization.

Security Services

Implementation guidance for AWS security services including:

  • AWS Identity and Access Management (IAM)
  • AWS Organizations
  • AWS Security Hub
  • Amazon GuardDuty
  • AWS Config
  • AWS CloudTrail
  • Amazon Macie
  • AWS IAM Access Analyzer
  • AWS Systems Manager
